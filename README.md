# Домашнее задание №1: Датасет русских юридических документов

## Описание проекта

В домашнем задании №1 представлен способ получения и обработки датасета юридических документов из открытых источников. Реализован парсер для легального сбора информации с бесплатных веб-порталов, содержащих нормативно-правовые акты Российской Федерации.

## Источники данных

Датасет состоит из следующих нормативно-правовых актов РФ:

1. Гражданский кодекс РФ (ГК РФ)
2. Трудовой кодекс РФ (ТК РФ)
3. Налоговый кодекс РФ (НК РФ)
4. Кодекс РФ об административных правонарушениях (КоАП)
5. Уголовный кодекс РФ (УК РФ)
6. Гражданский процессуальный кодекс РФ (ГПК РФ)
7. Уголовно-процессуальный кодекс РФ (УПК РФ)
8. Земельный кодекс РФ (ЗК РФ)
9. Жилищный кодекс РФ (ЖК РФ)
10. Арбитражный процессуальный кодекс РФ (АПК РФ)
11. Семейный кодекс РФ (СК РФ)
12. Бюджетный кодекс РФ (БК РФ)
13. Лесной кодекс РФ (ЛК РФ)
14. Градостроительный кодекс РФ (ГрК РФ)

## Методология

### Подход к сбору данных

Датасет собирается с открытых порталов, содержащего публично доступные нормативно-правовые акты. Это обеспечивает легальное получение информации в соответствии с политикой портала.

Поскольку информация с форумов по юридическим консультациям не распространяется свободно, парсинг ограничивается только нормативно-правовыми актами РФ (кодексами РФ).

### Парсинг данных

**Класс `BasicWebParser`** предоставляет следующий функционал:

- `fetch_page(url)` - загрузка страницы с обработкой ошибок и установкой корректных заголовков
- `parse_html(html_content)` - парсинг HTML контента используя BeautifulSoup
- `parse_site(url)` - комбинированная загрузка и парсинг страницы
- `extract_links(soup, base_url)` - извлечение всех ссылок со страницы с их текстом и атрибутами
- `extract_text(soup)` - извлечение текста контента из HTML с нормализацией
- `save_to_csv(data, filename)` - сохранение данных в CSV формат

**Особенности реализации:**
- Соблюдается вежливое поведение при парсинге: задержка между запросами (по умолчанию 1 сек)
- Использует реалистичные User-Agent строки для браузеров
- Правильная обработка кодировки (UTF-8)
- Логирование всех операций загрузки и ошибок
- Обработка исключений при сетевых ошибках

### Структура данных

Каждая статья сохраняется с полной иерархией документа:
- Название нормативно-правового акта
- Часть (если есть)
- Раздел (если есть)
- Подраздел (если есть)
- Глава
- Параграф (если есть)
- Статья
- Полный текст содержания

## Результаты анализа данных

### Распределение контента

Статьи распределены неравномерно по кодексам. **Гражданский кодекс РФ** содержит наибольшее количество статей в датасете.

### Структура выходных данных

На выходе получается DataFrame со следующими колонками:

| Колонка | Тип | Описание |
|---------|-----|---------|
| `url` | str | URL страницы статьи на портале base.garant.ru |
| `content` | str | Полный текст содержания статьи |
| `metadata` | dict | Справочная информация об иерархии (см. дополнительные типы) |

Дополнительные типы данных (используемые при анализе):

| Колонка | Тип | Описание |
|---------|-----|---------|
| `Название нормативно-правового акта` | str | Название кодекса (ГК РФ, ТК РФ и т.д.) |
| `Раздел` | str | Раздел документа (если применимо) |
| `Часть` | str | Часть документа (если применимо) |
| `Раздел` | str | Раздел документа (если применимо) |
| `Подраздел` | str | Подраздел документа (если применимо) |
| `Глава` | str | Номер и название главы |
| `Параграф` | str | Параграф (если применимо) |
| `Статья` | str | Номер и название статьи |

## Обработка данных

- Выявлены и удалены дубликаты контента (небольшие ошибки при парсинге)
- Классическая нормализация текста (NFKD, удаление лишних пробелов и переносов)
- Объединение данных из нескольких CSV файлов в единый DataFrame
